External Systems
Groq API
llama-3.3-70b-versatile, temperature=0. Called twice per turn: generate_sql (SQL drafting) and summarize (prose). Free tier — documented fallback, no org LLM gateway available.
MySQL 8 — retail_db
5 tables, no FK constraints (relationships are prompt-documented only in schema.py). Local instance, credentials via .env.
Read this as: the client never talks to Groq or MySQL directly — every external call is mediated by a node inside the graph, and every node reads/writes the same AgentState object. That single shared contract is what makes the conditional routing in graph.py possible (see the state-machine diagram for the exact control flow).
